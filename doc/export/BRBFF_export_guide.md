## 1. Concept

BRBFF replaces 3N Cartesian atom coordinates with a small set of rigid-body frame
poses plus fixed interpolation weights.  For a molecule like PTCDA (38 atoms, 108
internal DOFs), two frames at the bridge oxygens give 12 frame coordinates, of
which 6 are global rigid motion and 6 are internal bending/twisting/stretching
modes.  The internal 6×6 stiffness is fitted against a full atomistic Hessian or
energy/force samples.  An optional cubic+quartic polynomial extends the harmonic
model for finite-amplitude relaxation.

The key advantage is removing hard DOFs (bond stretches, angle bends) from the
dynamics, allowing much larger time steps since dt_max ~ sqrt(m/k) is limited by
the softest mode only.

### Core formula

```
x_i = w_{i0} (R_0 r_i + p_0) + w_{i1} (R_1 r_i + p_1)
```

where w_{i0} + w_{i1} = 1 (partition of unity).  Equal frame poses reproduce an
exact global rigid transform — this is the rigid-body invariance guarantee.

### Internal coordinate

```
eta = S * log(D_1^{-1} D_2),   S = diag(1,1,1, ell, ell, ell),   ell = |c2 - c1|
V = 1/2 eta^T K_eta eta + higher_order(eta)
```

## 2. Source Code Files

### 2.1 Core fitting and validation script

**File:** `topics/MultiGridFF/fit_stiffness.py` (~1100 lines)

This is the authoritative diagnostic and fitting script.  It:

1. Loads PTCDA, identifies bridge oxygens, computes smoothstep weights
2. Relaxes the molecule to the UFF minimum (L-BFGS-B)
3. Computes the UFF Hessian via finite differences
4. Builds the frame basis B (3N × 12) using **absolute positions** for rotation
5. Extracts the internal prolongation P_eta (3N × 6) via symmetric lift q1=-xi/2, q2=+xi/2
6. Fits stiffness via 5 methods:
   - **Galerkin (Ritz):** K = P_eta^T H P_eta — exact restriction, overestimates
   - **Spectral:** matches lowest UFF eigenvalues (dynamics only, needs mass)
   - **Edge-LS:** fits K_edge via C^T K_edge C ≈ K_f (12×12); limited for K=2
   - **E/Force fit:** samples UFF at random twists, fits K by linear LS on energy+force
   - **Relaxed (Schur):** static condensation, allows hard DOFs to relax
7. Fits a local cubic+quartic polynomial (ReducedPolynomialPotential) anchored to
   the Galerkin Hessian, with ridge regularization selected on held-out samples
8. Stabilizes the quartic term radially to ensure sampled positivity
9. Validates: virtual-work parity, damped dynamics, load/unload cycles, rigid invariance
10. Exports: anharmonic_model.npz, CSV files, PNG plots, log file

**Key functions defined here:**
- `rigid_basis_geometric(pos)` — 3 translations + 3 rotations about COM, QR-orthonormalized
- `build_frame_basis(pos, centers, weights)` — B matrix using absolute positions
- `build_internal_prolongation(B, pos, ell, masses, bMassOrthogonal)` — P_eta from symmetric lift
- `galerkin_stiffness(H, P_eta)` — K = P_eta^T H P_eta
- `spectral_stiffness(P_eta, masses, target_modes, target_eigenvalues)` — spectral calibration
- `fit_edge_stiffness_ls(K_f_12, C, lambda_reg)` — edge-sparse LS fit
- `generate_training_samples(...)` — UFF sampling with exact Jacobian-transpose forces
- `fit_stiffness_from_samples(samples)` — linear LS on energy+force
- `fit_stiffness_nonlinear_psd(samples, x0)` — PSD-constrained L-BFGS-B fit
- `relaxed_stiffness(H, pos, P_eta)` — Schur complement via pinv
- `relax_uff(eval_fn, pos0)` — L-BFGS-B relaxation to UFF minimum
- `validate_blended_frame_dynamics(...)` — virtual-work, damped dynamics, external-force tests
- `plot_energy_scans(...)` — per-DOF UFF vs harmonic vs anharmonic comparison
- `plot_anharmonic_validation(...)` — held-out parity scatter
- `plot_reduced_load_paths(...)` — quasistatic load/unload cycles

### 2.2 Library: Vibrations.py

**File:** `py/FFs/Vibrations.py` (~1143 lines)

Contains the reusable classes and functions that a target codebase needs:

- **`ReducedPolynomialPotential`** (dataclass): harmonic K + coordinate_scale + monomial powers + coefficients.  Methods: `energy_force(eta)`, `error_metrics(samples)`, `homogeneous_direction_range(degree)`, `save_npz(path)`, `load_npz(path)`.
- **`fit_reduced_polynomial(samples, K, min_degree, max_degree, coordinate_scale, ridge)`**: linear regression of higher-order monomials anchored to fixed K.  Uses `_monomial_values_gradients` for analytic gradients.
- **`stabilize_reduced_quartic_sampled(model, margin, n_directions, safety_factor)`**: adds isotropic ||eta/scale||^4 term to make sampled quartic positive.
- **`relax_reduced_potential(model, external_force, eta0, trust_radius, gtol)`**: L-BFGS-B with box constraints, then interior force-balance polish.
- **`relax_reduced_load_path(model, external_force_final, load_fractions, trust_radius)`**: quasistatic load continuation.
- **`reduced_potential_hessian(model, eta, relative_step)`**: central-difference Hessian from analytic forces.
- **`exp_se3(xi)`**: Rodrigues formula + SE(3) translation.  Inverse: `log_se3(D)`.
- **`reconstruct_blended_frame_positions(pos0, frame_transforms, weights)`**: skinning with absolute transforms.
- **`blended_frame_jacobian(pos0, frame_transforms, weights)`**: finite-difference Jacobian d(pos)/d(frame_increment).
- **`project_atomic_forces_to_frames(atom_forces, position_jacobian)`**: J^T F reduction.
- **`BlendedFrameDynamics`** (dataclass): full two-frame dynamics.  Methods: `evaluate(external_evaluator)`, `relax_step(external_evaluator, mobility, max_atom_step)`, `step_velocity_verlet(external_evaluator, dt, damping)`, `positions()`, `position_jacobian()`, `mass_matrix()`, `within_trust_region()`.
- **`relative_frame_eta(frame_transforms, length_scale)`**: extract eta = S * log(D1^{-1} D2) from current poses.
- **`relative_frame_eta_jacobian(frame_transforms, length_scale)`**: d(eta)/d(frame_increment).

Also contains standard vibration analysis: `hessian_from_ff`, `rigid_body_basis`, `project_rigid_modes`, `hessian_to_modes`, `atomic_masses`, `run_vibrations`.

### 2.3 GPU bridge: BRBFFF.py + BRBFFF.cl

**File:** `py/FFs/BRBFFF.py` (~291 lines) — Python OpenCL wrapper

**File:** `py/kernels/BRBFFF.cl` (~171 lines) — OpenCL kernels

The GPU path keeps atom positions, forces, and frame poses resident on device.  Three kernels:

1. **`brbfff_reconstruct_positions`**: skins atom positions from two quaternion frames + weights
2. **`brbfff_project_atomic_forces`**: reduces atom forces to frame wrenches (J^T F) via local-memory tree reduction
3. **`brbfff_relax_step`**: capped overdamped pose update with optional momentum smoothing

**Critical limitation:** The GPU path does NOT yet evaluate the fitted internal restoring potential.  It only propagates external forces.  An isolated distorted molecule will not return to equilibrium unless atom_force includes intrinsic molecular forces.

**Key design choices:**
- Quaternion-based rotation (not rotation matrices) for GPU efficiency
- One workgroup per molecule/system, local-memory reduction (no atomics)
- Partition-of-unity weights enforced at initialization
- Damping=0 gives pure capped gradient descent; damping>0 smooths successive steps
- Step history is NOT physical velocity or mass — just smoothing

### 2.4 Weight construction: blended_rigid_frames.py

**File:** `topics/MultiGridFF/blended_rigid_frames.py` (~187 lines)

Standalone script that loads PTCDA, identifies bridge vs carbonyl oxygens, places
frame centers, and computes two weight variants:
- **Radial exponential:** w ~ exp(-d/decay), renormalized
- **Linear smoothstep:** project onto c1→c2 axis, apply 3t²-2t³

Also plots the weight map with bonds, atoms, and frame centers.

### 2.5 Baseline vibration driver: run_vib_PTCDA.py

**File:** `topics/MultiGridFF/run_vib_PTCDA.py` (~90 lines)

Shows how to call `run_vibrations(mol, backend='uff')` and process modes/frequencies.

## 6. Critical Design Constraints (Do NOT Violate)

1. **Absolute positions for rotation arm**: `build_frame_basis` must use `r = pos[i]`, NOT `pos[i] - center[a]`.  Center-relative breaks rigid-body invariance when frames are blended.

2. **Partition of unity**: weights must satisfy `sum_a w_{ia} = 1` for every atom.  Otherwise equal frame poses do NOT reproduce a rigid transform.

3. **Coordinate scaling**: `eta = S * xi` where `S = diag(1,1,1,ell,ell,ell)`.  K acts on eta, not on raw xi.  Never mix K_eta with unscaled xi coordinates.

4. **Symmetric lift for prolongation**: `P_eta = 0.5 * (B2 - B1) @ S_inv`, then project out rigid modes.  This ensures `q2 - q1 = xi` exactly.

5. **Trust region for anharmonic model**: the cubic+quartic polynomial is a local Taylor fit.  Never extrapolate beyond `|eta_i| <= coordinate_scale_i`.  A boundary hit means out-of-domain, not a valid prediction.

6. **Float32 UFF noise floor**: the UFF evaluator uses float32 OpenCL.  Hessian stationarity has a ~1e-3 eV/Å force floor.  Do not trust inverse-Hessian condensation below this threshold.

7. **GPU has no internal restoring force**: `BRBFFF.cl` only propagates external atom forces.  The fitted relative-frame potential must be added separately (highest-priority TODO).

8. **Rigid mode removal before stiffness extraction**: always project out the 6 global rigid modes from B before computing K.  Use geometric rigid basis (translations + rotations about COM) for mechanics, mass-weighted for dynamics.

9. **Edge-LS limitation for K=2**: the incidence matrix C=[-I,I] cannot represent rigid-internal coupling.  For K=2, prefer Galerkin or E/F-fit.  Edge-LS becomes useful for K>=3 with a non-trivial frame graph.

10. **Paired training samples**: generate ±eta pairs for the anharmonic fit to eliminate odd-order bias and improve regression conditioning.

## 10. Glossary

- **BRBFF/BRBFFF**: Blended Rigid-Body Frame Force Field
- **Frame**: an SE(3) pose (rotation + translation) that controls a region of the molecule
- **Skinning**: reconstructing atom positions from frame poses via weighted blending
- **Partition of unity**: weights sum to 1 for each atom, ensuring rigid-body invariance
- **eta**: scaled relative twist between frames, the 6-dimensional internal coordinate
- **K_eta**: 6×6 symmetric stiffness matrix acting on eta
- **P_eta**: prolongation matrix mapping eta to Cartesian displacement (3N × 6)
- **Trust region**: the fitted coordinate box |eta_i| <= coordinate_scale_i where the polynomial model is valid
- **Galerkin**: exact restriction of Hessian to frame subspace (upper bound on stiffness)
- **Relaxed (Schur)**: static condensation allowing hard DOFs to relax (softer than Galerkin)
- **Edge-LS**: edge-sparse stiffness fit via incidence matrix (for frame graphs, K>=2)
- **Spectral**: match lowest UFF vibrational frequencies exactly (dynamics only)
- **E/F-fit**: energy/force least-squares fit from sampled UFF evaluations
